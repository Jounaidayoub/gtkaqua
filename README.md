# GTK4 Aquarium (C + GTK4)

A screensaver-style aquarium simulation written in C with GTK4.

This project uses sprite assets (`GtkPicture` widgets) and a fixed-timestep simulation loop to drive flocking, predator/prey behavior, eating, lifespan, and respawn.

## Project Layout

```
gtkaqua/
├── Makefile
├── include/
│   ├── config.h
│   ├── vec2.h
│   ├── species.h
│   ├── entity.h
│   └── world.h
├── src/
│   ├── main.c
│   ├── species.c
│   ├── entity.c
│   └── world.c
├── Fish/      # raw source assets
├── assets/    # prepared runtime assets (generated by make assets)
└── build/
    ├── obj/
    └── bin/aquarium
```

## Requirements

- GCC/Clang toolchain
- `make`
- GTK4 development packages (`pkg-config gtk4` must work)
- `ffmpeg` (for asset pipeline)
- `bear` (optional, for `compile_commands.json`)

## Quick Start

1. Prepare assets:

```bash
make assets
```

2. Build:

```bash
make
```

3. Run:

```bash
./build/bin/aquarium
```

## Build Targets

- `make` - build the app to `build/bin/aquarium`
- `make assets` - regenerate `assets/*.png` from `Fish/*`
- `make clean` - remove `build/`, `compile_commands.json`, and legacy root binary
- `make bear` - build with Bear and generate `compile_commands.json`

## Controls

- `F11` - toggle fullscreen
- `Esc` - exit fullscreen

## Architecture Overview

- `main.c`
  - GTK app/window setup
  - background CSS
  - key handling (F11/Esc)
  - fixed tick callback (`g_timeout_add`)
- `world.c`
  - world initialization
  - spawning/despawning
  - population setup
  - tick orchestration: update entities, eating, lifespan, visuals, respawn
- `entity.c`
  - boids forces (same-species separation/alignment/cohesion)
  - flee/hunt steering
  - same-species avoidance (config-driven)
  - movement integration and smooth rotation
  - sprite positioning and CSS transforms
- `species.c`
  - species registry (`SpeciesConfig` table)
  - movement/behavior/ecosystem tuning values

## Asset Orientation Contract

Runtime assumes all prepared sprites in `assets/` face LEFT.

`make assets` enforces this by flipping source fish that are right-facing.
If you replace raw sprites in `Fish/`, rerun `make assets`.

## Simulation Model (Current)

- Entity position is center-based in world space.
- Eating is mouth-based, not center-to-center:
  - mouth location is derived from heading and per-species `mouth_forward_ratio`.
- Entity lifecycle uses explicit states:
  - `ENTITY_ALIVE`, `ENTITY_DYING`, `ENTITY_INACTIVE`.
- Respawn:
  - species respawn to target `flock_size` with `respawn_delay`.

## Tuning Guide

Most gameplay tuning is in `src/species.c`.

Useful fields in `SpeciesConfig`:

- Movement: `max_speed`, `min_speed`, `max_force`
- Schooling: `separation_*`, `alignment_*`, `cohesion_*`
- Predator/prey: `fear_*`, `hunt_*`, `prey_mask`
- Predator spacing: `avoid_same_radius`, `avoid_same_weight`
- Eating geometry: `mouth_forward_ratio`
- Population: `flock_size`, `respawn_delay`, `lifespan`

## Notes for New Contributors

- Keep orientation fixes in the asset pipeline (`make assets`), not ad-hoc runtime flips.
- Prefer config-driven behavior in `SpeciesConfig` over species-specific hardcoded logic.
- Keep world position center-based; derive draw position in rendering code.
